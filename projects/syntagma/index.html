Syntagma

sourcedemoall projects

Agentic curriculum lifecycle management system for PES University. Faculty submit raw course content, the agent refines it into structured records, the 36-tool editor lets admins propose reviewable edits, and every state is snapshotable with full diff history and official A4 PDF output.

Designed and built the entire system: backend API, database schema, agent pipeline with 36 tools and type-checked tool registry, Jinja2 PDF templating engine, dual cache layer, CORS-proxied frontend on Vercel, and Docker backend on HF Spaces.

PES University revamps its B.Tech curriculum nearly every academic year. Faculty submit course content in raw, inconsistent formats. Someone manually compiles it into the official syllabus document. No version history, no way to compare what changed between years. Entirely manual, error-prone, slow.

Syntagma replaces that with an agentic lifecycle management system. Faculty submit through a form, the agent refines raw content into structured curriculum-ready records (Pydantic-validated, course codes auto-parsed for semester/department/credits). An agentic editor with 36 type-registered tools lets admins propose and review changes through reviewable drafts that never auto-apply. The curriculum renders as official A4 PDFs through a Jinja2 templating engine with four template variants (single course, semester, full curriculum, program outcomes) and multiple download options (inline preview, attachment download, single course, semester-level, full document). Every change is tracked through a git-like versioning system: named snapshots, restore, and version-vs-version diff.

The architecture runs on a 12-table Supabase PostgreSQL schema with dual cache layer (Upstash Redis with in-memory fallback and lazy pattern-based invalidation), cache prewarm at startup, and Sentry error tracking. The frontend is vanilla HTML/CSS/JS served from Vercel with API proxied through to the FastAPI Docker backend on Hugging Face Spaces. CORS middleware handles the cross-origin bridge. The agent tool system is built as a frozen dataclass registry with typed JSON Schema parameters, enabling complex tool chains: the agent can chain batch course reads, spreadsheet generation, web research, draft creation, and version snapshots in a single session. From competitor research to drafting courses to generating complete syllabuses in minutes.
