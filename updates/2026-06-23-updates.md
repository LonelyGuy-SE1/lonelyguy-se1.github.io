It has been a while since my last update, and quite a lot has happened since then.

For starters, I currently have multiple projects and experiments running in parallel. The largest one is still Raijin. Phase one of Raijin is now in a reasonably strong state. It is a fast limit order book engine built with consumer hardware in mind, and I have tried to incorporate as much relevant market microstructure and low-latency systems research as I could while building it.

More details are available on GitHub, along with the documentation and benchmark numbers:

![Raijin benchmark brief](gallery/bench.raijin.png)

The benchmark numbers shown here are a brief snapshot. The full benchmark numbers are available in the GitHub Actions summary.

GitHub: https://github.com/LonelyGuy-SE1/Raijin-LOB
Docs: https://lonelyguy-se1.github.io/Raijin-LOB/

The results so far are encouraging. The core performs well for what it is trying to be, and the benchmark numbers are available in the repository. I have now started phase two, which is the simulator. This part will take more time, since the research is still underway. The goal is to build a simulator that can replicate real market behaviour as closely as possible. I will need this for phase three, where I plan to move into the RL/JEPA side of the project.

Apart from Raijin, I have also been gradually building more experience with fine-tuning and small language models. The progress here has been steady. I am starting to get a much clearer intuition for where small models are useful, where they break, and what kind of data actually improves them.

One of my recent releases is Project Halide, a diagnostic workbench for film negatives. Since this was built as a hackathon submission, I tried to make it as practically useful as possible rather than treating it as a toy demo. The hardest part was the data. There are almost no reliable public datasets for this specific problem, so I had to create my own synthetic data and fine-tune the vision model around that. The fine-tuning led to a noticeable improvement, and I have written a separate report/blog about the project in the blog section.

Blog: https://lonelyguy.tech/articles/2026-06-16-project-halide

I also built a small Chrome extension to track and auto-sync my anime watchlist with an anime site. There is not much to say about this one, since it is mostly a personal utility, but it was useful enough for me to keep.

![Shiori extension interface](gallery/shiori.extension.png)

This week, I also spent some time working with core Android system files and managed to fix a few issues at the OS level for my custom ROM devices.

On the competitive programming side, I participated in my first couple of CP contests. I am still early here, but I have started treating it as a regular skill-building process rather than something occasional.

I am also working on a fast in-memory database, along with a set of general AI experiments. Some of these involve conceptual learning, paper implementations, and older architectures such as ResNet and LSTM. A lot of this is still exploratory, but it has been useful for improving my understanding from first principles.

I have also been spending a significant amount of time on simulations using ROS 2, MuJoCo, MATLAB, and related tools. Most of this work is still undocumented because it is experimental and difficult to present cleanly. If the Hawkes simulator progresses well, I plan to intensify my work on robotics-related simulation projects as well.

That is mostly what I have been working on since the previous update. I might also attend a few research-oriented events if time permits, but for now the main priority is to keep building and documenting the work that is already underway.
